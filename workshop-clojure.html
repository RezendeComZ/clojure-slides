Clojure

Um Lisp moderno para a JVM (e além!)

O que é Clojure?

  • Uma Linguagem Dinâmica (como Python ou Ruby)
  • Uma Linguagem Funcional (ênfase em funções e imutabilidade)
  • Um Dialeto Lisp (código é dado, sintaxe simples)
  • "Hospedada" (roda na JVM, Javascript, .NET, etc.)
  • Focada em Simplicidade e Concorrência

O Oposto de "Fácil"

"Fácil" (Easy)

  • Significa "próximo", "familiar".
  • Ex: Um IDE com 1000 botões. É fácil de começar a clicar.
  • Muitas vezes esconde complexidade.

"Simples" (Simple)

  • O oposto de "complexo".
  • Significa "uma coisa só", "desentrelaçado".
  • Ex: Um git commit na linha de comando. É simples (faz uma coisa), mas não é fácil de aprender no início.
  • Clojure foca em simplicidade.

Pilar 1: Imutabilidade por Padrão

Em Clojure, dados não mudam. Você cria novos dados.

  • [1 2 3] é um vetor.
  • (conj [1 2 3] 4) não modifica o vetor original.
  • Ele retorna um novo vetor: [1 2 3 4].
  • O original [1 2 3] permanece intacto.

Por quê?

  • Código previsível (a função não vai mudar seus dados).
  • A chave para concorrência segura e fácil!

Estruturas de Dados Literais

;; Vetor (acesso rápido por índice)
[1 "olá" :keyword 3.14]

;; Mapa (chave-valor)
{:nome "Ana" :idade 30 :skills [:clojure :java]}

;; Lista (S-expression, "Code as Data")
'(+ 1 2) ; dados

;; Set (valores únicos)
#{:a :b :c :a} ; => #{:a :b :c}
